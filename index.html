HTML

  • The head element contains information about the webpage.
  • The head element is a container for metadata.
  • Meta elements contain information about the page that is used by the browser.
  • The body element represent the visible content shown to the user.
  • HTML stands for Hypertext Markup Language, which gives the website its structure.
  • The name of semantic elements describes their function and should be used as much as possible, to keep the code easy to read
  • Elements are a type of content on a webpage.
  • HTML elements are case sensitive, whereas declerations are not.
  • An HTML attribute defines additional information about an element, and are always defined in the opening tag.
  • The head element contains the metadata, which the browser reads to understand the website.
  • Title elements are extremely important for Search Engine Optimization (SEO)
  • The meta element defines metadata such as the character set, descriptiom, keywords, author, and viewport.
  • Metadata is data about data.
  • The link element connects and external resource to the HTML element.
  • The charset element sets the character encoding
  • UTF-8 is backwards compatible with ASCII and the preferred encoding for e-mail and web pages.
  • The http-equiv element sets an HTTP header for content.
  • Elements nested inside other elements are called child elements.
  • The element that holds the children is called the parent element.
  • The body element defines the body of the document, which is visible to the user
  • Everything that needs to be shown to the user must be in the body element.
  • The div element creates a division in the page
  • The span element creates an inline division
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what apperats inside the angle brackets.
  • The p element represents a paragpraph or block of text.
  • The ul element represents unordered list
  • The ol element represents ordered list
  • The li element represents list elements
  • The a (anchor) element creates link to the same or other webpages.
  • Header tags from h1 to h6, while h1 should only be used once on each page.
  • The src attribute defines the location, or source of the image file.
  • The alt attribute contains a text string that describes the appearance and functionality of the image.
  • Alt text can also improve search engine rankings for the website.
  • The section elements are semantic elements that define a section in a document that contains thematic content

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element to which declerations will apply.
  • Declerations contain two important components: property and value
  • The margin property indicates how much space there is around the outside of an element.
  • The padding property adds space around the content, inside an element.
  • Increasing the padding value increases the size of the entire element.
  • The * selector targets all elements visible on the page.
  • Do No Repeat Yourself (DRY)
  • The rel attribute in the link element specifies the relationship between the current document and the linked document resource.
  • The href attribute in the link element specifies the URL of the external resource.
  • href is using a relative path, meaning location is derived from the current location.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • When naming a repository use lowercase letters and dashes as spaces
  • MIT license
  • README.md is a markdown file
  • GitHub issues are like a to-do list inside the GitHub repository.
  • git clone "paste copied SSH value": clones the GitHub repository onto the local environe
  • "code .": opens VS Code in the current directory.
  • A pull request, or PR, is used to merge a cmoparison branch into the base branch.
  • The base/main branch is where changes will be merged into.
  • checkout: move the working branch to a new branch.
  • -b: creates a new branch.
  • git add -A: 'add' adds changes in the current working branch to the staging area. '-A' stands for adding all changes.
  • git pull origin main: pull command to receive a branch's changes into the local environment. 'origin', stands for the source of the pull will come from the GitHub repo. 'main' indicates the branch.
  • git push origin "name of branch": 'push' to push local changes to the remote GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow refers to the order in which a computer executescode based on our instructions
  • We can instruct a program to deviate from the normal linear flow of an application, if certain conditions are met
  • We can use conditional statements and loops to control the order in which the code is executed.
  • An array is useful for storing multiple items with different data types.
  • An index defines the location of a data item in the array. JavaScript is 0-indexed. (start from 0)
  • console refers to a test environment that checks the code.
  • .log() will output whatever is inside the parentheis.
  • The script element with the source attribute links the JavaScript file to the HTML.
  • The script element goes just before the closing body tag.
  • When using if statements, the code will only execute if the statement meets a condition, or is true.
    • else if for multiple conditions.
  • "===" is a strict equality operator and checks if two values are equal and returns a boolean expression.
  • A for loop contains three important statements.
    • The first statement defines the starting point for our loop. (x=0)
    • The second statement is the condition. (x < array.length)
    • The third statement allows the array to iterate over each item. (x++)
  • To define a function, we use the function keywork, followed by the function name with parenthesis and the function block.
  • /* ... */ adds comments in JavaScript.
